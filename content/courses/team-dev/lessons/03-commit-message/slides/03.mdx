# Conventional Commits の型

多くのチームは、先頭に「変更の種類」を付けるルールを使っています。

```
<type>: <日本語の要約>
```

`type` によく使うものは次のとおりです(英語で書きます)。

- `feat` — 新しい機能を追加した
- `fix` — バグを直した
- `docs` — ドキュメントだけを変えた
- `refactor` — 動きは変えずに中身を整理した
- `test` — テストを追加・修正した
- `chore` — 設定やビルドなど、その他の雑務

<Callout type="info">
type は英語、要約は日本語で書くのが、日本語チームでの現実的な運用です。要約は 50 文字くらいまでを目安に、簡潔に。
</Callout>
