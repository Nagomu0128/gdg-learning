# PR 説明文の型

多くのチームは、次のような見出しをテンプレートにしています。

```
## 目的
なぜこの変更をするのか

## 変更点
- 何を変えたか(箇条書き)

## テスト
どう動作確認したか

## レビュー依頼
とくに見てほしい点
```

- **目的** — 背景や、解決したい問題を書く
- **変更点** — 変えた内容を箇条書きで並べる
- **テスト** — 自分で確認した手順や結果を書く
- **レビュー依頼** — 迷った点や、重点的に見てほしい点を伝える

<Callout type="info">
見出しは Markdown の `##` で作ります。箇条書きは行の先頭に `- ` を付けます。
</Callout>
