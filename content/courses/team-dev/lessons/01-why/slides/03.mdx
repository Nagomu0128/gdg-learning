# バージョン管理がチームを支える

Git のようなバージョン管理は、変更を「履歴」として積み重ねます。これがチーム開発の土台です。

- **履歴** — だれが・いつ・何を変えたかが全部残り、前の状態にも戻せる
- **コンフリクト** — 2人が同じ場所を変えたときに起きる「ぶつかり合い」。Git はこれを見つけて教えてくれる
- **ルール** — コミットの書き方やブランチの使い方を、チームでそろえておく決めごと

<Callout type="info">
このコースでは、ブランチ戦略・コミットメッセージ・プルリクエスト・レビューといった「チームのルール」を 1 つずつ学んでいきます。
</Callout>
