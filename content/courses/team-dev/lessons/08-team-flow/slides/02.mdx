# 開発の一巡

チーム開発は、だいたいこの流れをぐるぐると回します。

```
Issue      やることを決める(粒度・受け入れ条件)
  ↓
ブランチ    main から feature ブランチを切る
  ↓
コミット    小さく変更し、Conventional Commits で記録
  ↓
PR         変更をまとめてレビューを依頼する
  ↓
レビュー    指摘をやりとりして、コードを良くする
  ↓
マージ      main に取り込む(main は動くまま)
  ↓
片付け      使い終わった feature ブランチを消す
```

<Callout type="info">
このシミュレータには PR やレビューの画面はありません。今回は「レビューが通った」というつもりで、ブランチ・コミット・マージ・片付けのコマンド部分を練習します。
</Callout>
