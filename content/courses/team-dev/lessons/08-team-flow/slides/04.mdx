# やってみよう

`commands.sh` に、開発の一巡を上から順に書きます。

- main はさわらず、まず feature ブランチを作って作業します
- コミットメッセージは Conventional Commits(`feat:` など)で
- レビューが通ったつもりで main に戻り、マージして、feature ブランチを片付けます

うまくいくと、main にあなたのコミットが取り込まれ、feature ブランチは消えて、きれいな状態になります。

<Callout type="info">
これで「チーム開発入門」は修了です。ここで身につけた流れが、実際のチーム開発の土台になります。おつかれさまでした。
</Callout>
