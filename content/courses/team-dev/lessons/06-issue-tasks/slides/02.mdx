# 大きすぎる Issue、小さすぎる Issue

Issue の大きさは、大きすぎても小さすぎても進めにくくなります。

- **大きすぎる** — 「サイトを作る」。何から手を付けるか分からず、いつ終わるかも見えない
- **小さすぎる** — 「セミコロンを1つ足す」。管理する手間のほうが大きい
- **ちょうどよい** — 「問い合わせフォームに送信結果の表示を追加する」。1 つの PR で片付く単位

<Callout type="info">
目安は「1 つの feature ブランチ、1 つの PR でおさまるくらい」。大きいと感じたら、複数の Issue に分けましょう。
</Callout>
