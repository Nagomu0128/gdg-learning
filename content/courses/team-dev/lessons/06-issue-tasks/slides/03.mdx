# 受け入れ条件とタスク分解

良い Issue には「これが満たせたら完了」という受け入れ条件があります。ゴールが決まると、迷わず進められます。

```
## 背景
送信後に何も表示されず、送れたか分からないという声がある

## 受け入れ条件
- [ ] 送信に成功したら「送信しました」と表示される
- [ ] 送信に失敗したら理由が表示される

## タスク
- [ ] 成功時のメッセージ表示を追加する
- [ ] 失敗時のエラー表示を追加する
```

- **受け入れ条件** — 「どうなったら終わりか」を、確認できる形で書く
- **タスク** — その条件を満たすためにやることを、小さく分ける
- `- [ ]` はチェックボックス。終わったら `- [x]` にして進み具合を共有できる

<Callout type="info">
条件とタスクは似ていますが、条件は「ゴールの状態」、タスクは「そこへ行くための作業」です。
</Callout>
