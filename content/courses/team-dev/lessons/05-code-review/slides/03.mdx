# must-fix と nit を分ける

指摘には「必ず直してほしいもの」と「直さなくてもよい好みの提案」が混ざります。これを区別すると、相手は優先順位を付けやすくなります。

- **must-fix** — 必ず直したい指摘。バグやセキュリティの問題など
- **nit** — nitpick(細かい指摘)の略。命名やスタイルなど、好みレベルの軽い提案。直さなくてもよい

そして、指摘を受ける側にも作法があります。

- 指摘してくれたことに、まず感謝する
- 直す・直さないを、理由を添えて返す
- 分からなければ質問する。ため込まず対話する

<Callout type="info">
「nit:」と一言そえるだけで、相手は「これは軽い提案なんだ」と分かります。小さな気配りが、レビューを気持ちよくします。
</Callout>
