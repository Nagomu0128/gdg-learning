# 指摘は「具体的」で「敬意をもって」

同じ内容でも、言い方で受け取られ方が変わります。

```
これはダメ。            ← 何がどうダメか分からない/責める口調
なんでこう書いたの?      ← 問い詰めているように聞こえる
```

具体的に、提案の形で伝えると、受け取る側も動きやすくなります。

```
ここは null のとき落ちそうです。先に空チェックを入れると安全だと思います。
変数名を users にすると、中身が配列だと伝わって読みやすくなりそうです。
```

<Callout type="warn">
「あなたが間違っている」ではなく「コードをこうするともっと良くなる」。主語をコードに向けると、指摘が伝わりやすくなります。
</Callout>
