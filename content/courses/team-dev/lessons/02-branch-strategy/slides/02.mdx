# GitHub Flow: main は常に動く

GitHub Flow は、とてもシンプルなブランチの使い方です。ルールはたった 3 つ。

- **main は常に動く** — main にあるコードは、いつ見ても動く状態にしておく
- **作業は feature ブランチで** — 新しい機能や修正は、main から枝分かれさせた専用ブランチで進める
- **小さく進める** — 1 ブランチ = 1 つの目的。大きくしすぎず、こまめに区切る

<Callout type="warn">
main で直接作業すると、まだ途中のコードがチーム全員に影響します。だから「枝分かれ(ブランチ)」で自分の作業を隔離するのです。
</Callout>
