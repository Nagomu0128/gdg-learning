# CD と、main を守るという考え方

CD(Continuous Delivery / 継続的デリバリー)は、テストを通ったコードを、自動で利用者のところへ届ける(デプロイする)仕組みです。

そして CI/CD の土台にあるのが「main を守る」という考え方です。

- テストが通らないコードは、main に取り込めないようにする
- レビューが済んでいない変更も、取り込めないようにする
- こうして main を「いつでも公開できる状態」に保つ

<Callout type="warn">
main がこわれると、そこから枝分かれする全員に影響します。だから「取り込む前に必ずチェックを通す」というルールを、自動の仕組みで守るのです。
</Callout>
