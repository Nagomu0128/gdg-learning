# コンフリクトの印

同じ行を両方が変えていると、Git はそのファイルに「両方の案」を並べて書き込み、手が止まります。

```
<<<<<<< HEAD
こんにちは
=======
こんばんは
>>>>>>> feature
```

- `<<<<<<< HEAD` から `=======` まで … 今いるブランチ(main)側の内容
- `=======` から `>>>>>>> feature` まで … 取り込む側(feature)の内容

<Callout type="warn">
この 3 種類の印(`<<<<<<<` `=======` `>>>>>>>`)は、そのままにしてはいけません。印ごと消して、最終的に残したい内容だけにするのが「解決」です。
</Callout>
