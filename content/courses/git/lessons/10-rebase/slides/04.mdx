# やってみよう

`commands.sh` に、リベースのコマンドを 1 行書きます。

```
git rebase main
```

うまくいくと

```
feature を付け替えました(rebase 完了: 1 個のコミットを再適用)
```

と表示されます。`git log --oneline` を足すと、main の緊急修正の「上に」自分の機能コミットが乗り、履歴が 1 本の線になっているのが分かります。

<Callout type="info">
main と feature が別々のファイルを触っているので、今回は衝突なしで載せ替わります。同じ場所を触っていると衝突します。その解決は次のレッスンで学びます。
</Callout>
