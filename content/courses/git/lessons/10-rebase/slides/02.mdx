# 載せ替えのイメージ

feature で作業している間に、main も進んでしまいました。

```
before:
        ●───◇ feature(機能1)← 今ここ
       /
●───●───▲ main(緊急修正)
```

リベースは、feature のコミット(◇)をいったん外し、main の先端(▲)の続きとして置き直します。

```
after:
●───●───▲───◇ feature(main の続きになった)
```

<Callout type="info">
◇ の中身は同じですが、「どのコミットの続きか(親)」が変わるので、新しいコミットとして作り直されます。だから履歴が 1 本にそろいます。
</Callout>
