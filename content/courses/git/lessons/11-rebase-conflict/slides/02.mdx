# なぜ rebase でも衝突する?

リベースは、自分のコミットを相手の先端に「置き直す」動きでした。置き直すとき、同じ場所を相手も変えていると、Git は自動で決められません。

```
feature: config.txt を「機能版」に変更
main:    config.txt を「本番版」に変更
```

- 両方が同じ `config.txt` を変えている
- feature のコミットを main の上に載せ直そうとした瞬間、衝突が起きる
- ファイルには merge のときと同じ `<<<<<<<` の印が入ります

<Callout type="warn">
リベース中は「載せ替えの途中」という特別な状態です。`git status` を見ると「リベースの途中です」と教えてくれます。
</Callout>
