# リモートが先に進むと

あなたが手元で作業している間に、他の人が push して、リモートが先に進むことがあります。

```
リモート(origin/main)  ●───●───●   ← 他の人の更新で先に進んだ
手元(main)            ●───●        ← 少し遅れている
```

- 手元は「1 つ前」の状態
- このまま push しようとすると「先に進んでいるから、先に取り込んで」と断られます
- だから、まずリモートの更新を受け取って追いつきます

<Callout type="info">
「push が断られた」ときは、たいてい「先に pull して」のサインです。あわてず、まず受け取るのが基本の順番です。
</Callout>
