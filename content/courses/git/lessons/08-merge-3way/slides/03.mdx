# 衝突しないマージ

2 つのブランチが**別々のファイル**を変えているなら、Git は自動でうまく合わせてくれます。

- feature は `feature.txt` を追加
- main は `docs.txt` を追加
- 触っている場所が違うので、機械的に両方を採用できます

```
git merge feature
```

書き方は早送りのときと同じ、`git merge ブランチ名` の 1 行です。違いは「結果としてマージコミットができる」ことだけです。

<Callout type="warn">
同じファイルの同じあたりを両方が変えていると、Git は自動で決められず「コンフリクト(衝突)」になります。その解決は次のレッスンで学びます。
</Callout>
