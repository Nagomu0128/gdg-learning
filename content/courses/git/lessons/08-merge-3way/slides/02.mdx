# 両方が進むとどうなる?

早送りと違い、今回は main も feature も枝分かれのあとに進んでいます。

```
         ●───● feature(機能A を追加)
        /
●───●          main(ドキュメントを追加)← 今ここ
```

- main を「進めるだけ」では feature の変更を取り込めません
- Git は両方の変更を見比べて、1 つに合わせた新しいコミットを作ります
- これが**マージコミット**です

```
         ●───●
        /     \
●───●───●──────◆  ← マージコミット(親が2つ)
```

<Callout type="info">
◆ のマージコミットは、左右 2 つの流れを親に持ちます。「ここで 2 つの歴史が 1 つになった」という記録です。
</Callout>
