# 早送り(fast-forward)とは

取り込む側(main)が枝分かれのあと 1 度もコミットしていないとき、マージはとても簡単になります。

```
取り込み前:
main     ●            ← 今ここ(動いていない)
          \
feature    ●───●      ← 先に進んでいる

取り込み後:
main     ●───●───●    ← main を feature の先端まで「進めるだけ」
```

- main は枝分かれ以降 進んでいないので、**新しいコミットは作りません**
- ただ main の名札を、feature の先端まで前に動かすだけ
- これを「早送り(fast-forward)」と呼びます

<Callout type="info">
早送りでは履歴が 1 本の線のまま。枝分かれの跡が残らないので、いちばんスッキリした合流です。
</Callout>
