# git status — いまの状態

`git status` は「次のコミットに向けて、いま何がどうなっているか」を教えてくれます。

- **ステージ済み**の変更(commit すると記録されるもの)
- **ステージされていない**変更(まだ add していないもの)
- **未追跡**のファイル(まだ一度も add していないもの)

作業ツリーがきれいなら、こう表示されます。

```
コミットするものはありません。作業ツリーはきれいな状態です
```

<Callout type="info">
「あれ、今どうなってたっけ?」と思ったら status。add / commit の前後で状態がどう変わるかを確かめる習慣をつけましょう。
</Callout>
