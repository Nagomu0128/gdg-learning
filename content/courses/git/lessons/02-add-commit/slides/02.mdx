# コミットは「積み重ね」

Git の履歴は、コミットを 1 つずつ積み上げてできています。新しい変更を残したいときは、そのたびに add と commit をします。

- **add** — 記録したい変更を「ステージ」という待機場所に載せる
- **commit** — ステージの内容を 1 つの記録(セーブポイント)にする

```
変更をつくる ── git add ──▶ ステージ ── git commit ──▶ 履歴が1つ増える
```

<Callout type="info">
「変更したら必ず記録される」わけではありません。add して commit した変更だけが、履歴に残ります。
</Callout>
