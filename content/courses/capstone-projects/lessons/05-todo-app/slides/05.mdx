# やってみよう

完成条件(= 判定が見るもの)は次の4つです。

1. `#todo-input` に文字を入れて `#add-button` を押すと、`#todo-list` に li が増える(入力した文字が項目に入る)
2. li をクリックすると `class="done"` が付き、もう一度クリックすると外れる
3. li の中の削除ボタン(`class="delete"`)を押すと、その li が消える
4. `#count` の数字が「未完了の件数」といつも一致している

全部そろったら、自由に育てましょう。アイデアの例:

- エンターキーでも追加できるようにする(input の keydown イベントで、event.key が "Enter" のときに追加)
- 完了した項目の見た目を工夫する(取り消し線・うすい色・チェックマーク)
- 「全部削除」ボタンや、完了済みの件数の表示を足す

<Callout type="warn">提出前に、追加 → 完了 → もう一度クリックで戻す → 削除、の操作をプレビューで全部たしかめ、そのたびに件数が正しく変わることを見ておきましょう。</Callout>
