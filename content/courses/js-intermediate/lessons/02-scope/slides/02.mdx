# 関数スコープ

関数の中で宣言した変数は、その関数の中だけで使えます。外からは見えません。

```js
function sayHello() {
  const message = "こんにちは";
  console.log(message); // ここでは使える
}

sayHello();
console.log(message); // エラー! 関数の外からは見えない
```

逆に、関数の外(グローバル)で宣言した変数はどこからでも見えます。便利に思えますが、どこからでも書き換えられるため、思わぬバグのもとになります。

<Callout type="info">変数は「使う場所にいちばん近いところ」で宣言するのが基本です。関数の中だけで使う変数は、関数の中で宣言しましょう。</Callout>
