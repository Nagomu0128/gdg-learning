# 状態と表示を分ける

カウンターの「今の数」はどこにあるべきでしょうか。画面の文字そのものではなく、JavaScript の変数で持つのがコツです。

```js
let count = 0; // これが「状態」

function render() {
  display.textContent = count; // 状態を画面に写す
}
```

どのボタンの処理も、次の 2 段で書けます。

1. 変数を変える(`count = count + 1` など)
2. 表示を書き換える(`render()` を呼ぶ)

<Callout type="info">「本当の値は変数、画面はその写し」という分け方は、前に作った配列からのリスト描画と同じ発想です。大きなアプリでもこの形は変わりません。</Callout>
